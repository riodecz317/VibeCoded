Usage Guide
How the System Works
The Kanban Board and Performance Review tool share data through browser storage. Each employee owns one board; the manager collects bundles.
1️⃣
Employee Sets Up Kanban
Opens kanban-connected.html in their browser. Clicks Owner: field in the header and types their full name. This name is their identity in the performance review system.
2️⃣
Employee Works & Exports
Works on their board throughout the quarter. When ready, clicks 📤 Send to Manager in the header. A KanbanBundle_Name_Date.json file is downloaded and sent to the manager.
3️⃣
Manager Imports & Reviews
Opens the Kanban Data tab → 📥 Import Manager Bundle. The employee appears automatically. Click ＋ Review on their board card to open a pre-filled review form.
⚠ Important — Owner Name = Employee Identity
The Owner name set in the Kanban board header is the employee's identity across the entire system. All performance data, reviews, and headcount counts are linked to this name.
• The name must be set before using 📤 Send to Manager
• Names are case-sensitive — "Maria Santos" and "maria santos" are two different employees
• If the name changes, existing reviews must be manually updated

How Kanban Scores Are Derived
Kanban board activity is automatically converted into 1–5 performance scores
ScoreDerived from KanbanFormula
Task CompletionCards moved to Done columns1 + (done ÷ total) × 4
Work QualitySubtask completion rate1 + (subtasks done ÷ total subtasks) × 4
PunctualityCards with past due dates not completed5 − (overdue ÷ dated cards) × 4
InitiativeHigh-priority card handling rate1 + min(high-pri rate × 1.5, 1) × 4
Criteria linked to a Kanban source are 🔗 auto-filled in the review modal. All scores remain manually editable — Kanban data is a starting point, not a final grade.
